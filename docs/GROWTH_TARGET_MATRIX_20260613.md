## Immediate Send Packet

### Provider / Publisher Ask

Subject: BuilderPerks developer inventory pilot

Hi,

BuilderPerks is testing a developer-native sponsored offer surface for terminals, IDEs, browser extensions, status lines, and AI-agent UIs.

The live API lets a publisher register a surface, send broad programming keywords only, and render one labeled sponsored devtool offer. It tracks impressions, clicks, claims, and relevance actions like Need this / Not relevant / Hide category.

We are not looking to pass through unapproved network demand. We are looking for the right approved partner path for developer inventory once we have enough supply proof.

Live site: `https://builderperks.netlify.app/`
Sample stream: `https://builderperks.netlify.app/api/ad-stream?publisherId=pub_x&surface=terminal&context=deploying%20an%20AI%20app&keywords=typescript,react,postgres&format=statusline`

What would you need to evaluate a small pilot or publisher/ad-serving partnership?

Thanks,
Barney
