Supply-side API

Stream a sponsored card into a terminal, IDE, extension, or agent UI.

Founding publishers can register a surface, call one endpoint, and receive a labeled sponsored offer with a tracked click URL. Earnings are estimated until advertiser revenue and payout rails are approved, then can help offset AI and developer tooling bills.

1. Register a publisher surface.
2. Install the status-line helper with one command.
3. Run it from your terminal, IDE, Claude Code status line, or agent UI.
curl -X POST https://builderperks.netlify.app/api/publishers \
  -H "content-type: application/json" \
  -d '{"name":"My CLI","email":"founder@example.com","surface":"terminal"}'
curl -fsSL https://builderperks.netlify.app/install-statusline.sh | bash -s -- pub_x
BUILDERPERKS_PUBLISHER_ID=pub_x BUILDERPERKS_KEYWORDS=typescript,react,postgres BUILDERPERKS_BLOCKED_KEYWORDS=crypto,gambling BUILDERPERKS_BLOCKED_CATEGORIES=adult,gambling BUILDERPERKS_VALUE_MODE=relevant ~/.builderperks/statusline.sh
curl "https://builderperks.netlify.app/api/ad-stream?publisherId=pub_x&surface=terminal&context=deploying%20an%20AI%20app&keywords=typescript,react,postgres&blockedKeywords=crypto,gambling&blockedCategories=adult,gambling&valueMode=relevant&format=statusline"

For targeting, send broad programming language, framework, and project keywords only. Use valueMode for passive, relevant, or high-value discovery. Use allowedCategories for explicit opt-ins and blockedCategories for ad groups you do not want in your workflow. Do not send personal data or full prompts. The helper fails quietly if the network or API is down, so it will not break a prompt, terminal, or status-line workflow. Terminal publishers can use `format=statusline` and render the returned `render.statusLine` value directly. IDE and agent UIs can use `render.ideCard`; Markdown surfaces can use `render.markdown`.
