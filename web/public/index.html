Supply-side API

Stream a sponsored card into a terminal, IDE, extension, or agent UI.

Founding publishers can register a surface, call one endpoint, and receive a labeled sponsored offer with a tracked click URL. Raw impressions and estimated earnings are directional only until proof review, advertiser revenue, and payout rails are approved.

1. Register a publisher surface.
2. Install the status-line helper with one command; it saves your publisher id and token locally.
3. Run ~/.builderperks/statusline.sh from your terminal, IDE, Claude Code status line, or agent UI.
4. Verify setup with BUILDERPERKS_DEBUG=1 ~/.builderperks/statusline.sh if the line is blank.
curl -X POST https://builderperks.netlify.app/api/publishers \
  -H "content-type: application/json" \
  -d '{"name":"My CLI","email":"founder@example.com","surface":"terminal"}'
curl -fsSL https://builderperks.netlify.app/install-statusline.sh | bash -s -- pub_x bp_pub_token
curl -fsS https://builderperks.netlify.app/install-statusline.sh -o install-statusline.sh
less install-statusline.sh
bash install-statusline.sh pub_x bp_pub_token
~/.builderperks/statusline.sh
BUILDERPERKS_DEBUG=1 ~/.builderperks/statusline.sh
"statusLine": { "type": "command", "command": "~/.builderperks/statusline.sh" }
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "BuilderPerks sponsored line",
      "type": "shell",
      "command": "~/.builderperks/statusline.sh",
      "problemMatcher": []
    }
  ]
}
curl "https://builderperks.netlify.app/api/ad-stream?publisherId=pub_x&publisherToken=bp_pub_token&surface=terminal&context=deploying%20an%20AI%20app&keywords=typescript,react,postgres&blockedKeywords=crypto,gambling&blockedCategories=adult,gambling&valueMode=relevant&format=statusline"

For targeting, send broad programming language, framework, and project keywords only. Use valueMode for passive, relevant, or high-value discovery. Use allowedCategories for explicit opt-ins and blockedCategories for ad groups you do not want in your workflow. Current category lanes include devtools, cloud, AI, security, data, productivity, design, marketing, ecommerce, education, jobs, travel, real estate, finance, legal, health, gaming, crypto, adult, dating, sweepstakes, and gambling/iGaming. The installer writes ~/.builderperks/config.env with your publisher id, publisher token, and safe defaults. You can still override a single run with variables such as BUILDERPERKS_VALUE_MODE. Do not send personal data or full prompts. The helper fails quietly if the network or API is down, so it will not break a prompt, terminal, or status-line workflow. Use BUILDERPERKS_DEBUG=1 only when verifying setup; diagnostics go to stderr. Terminal publishers can use `format=statusline` and render the returned `render.statusLine` value directly. IDE and agent UIs can use `render.ideCard`; Markdown surfaces can use `render.markdown`. VS Code and Cursor can run the helper as a shell task, so the first integration is a command users already understand.
